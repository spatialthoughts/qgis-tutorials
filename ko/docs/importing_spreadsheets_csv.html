스프레드쉬트 혹은 CSV파일 불러오기¶

경고

아래 새로운 버전에 맞춘 GIS 튜토리얼을 참고하시길 바랍니다. 스프레드시트 및 CSV 파일 불러오기 (QGIS3)

많은 경우 GIS 데이터는 표나 엑셀 스프레드쉬트로 접하게 됩니다. 만약 위도/경도좌표 리스트가 있는 경우에도 GIS프로젝트에서 쉽게 불러올 수 있습니다.

작업 개요¶

텍스트로 되어 있는 지진 데이터를 QGiS로 불러옵니다.

데이터 획득¶

NOAA의 국가지리데이터센터는 기원전 2150년전부터의 방대하고 중요한 모든 지진데이터셋을 보유하고 있습니다. Learn more.

Download Significant Earthquake Database text file.

아래의 링크에서 데이터를 내려받을 수 있습니다.

signif.txt

Data Source [NCEI]

과정¶

  1. 테이블 형태의 테이터를 살펴보십시오. 이 데이터를 QGIS로 읽어 들여오기 위해서는 텍스트파일로 저장을 해야합니다. 그리고 적어도 X, Y좌표가 포함된 최소 2칼럼이 필요합니다. 스프레드쉬트파일이라면 해당 프로그램의 ‘다른 이름으로 저장 Save As’기능을 이용해서 Tab Delimited File 혹은 `Comma Separated Values (CSV)`로 저장을 합니다. 일단 이런 식 내보낸 데이터를 가지고 있으면 노트패드와 같은 텍스트 편집프로그램으로 파일을 열어서 내용을 살펴볼 수 있습니다. 중요지진데이터의 경우에는 이미 다른 속성데이터와 더불어 지진센터의 경위도가 포함된 텍스트파일입니다. 각 필드는 탭으로 분리되어 있음을 알 수 있습니다.

  1. QGIS를 열고 :menuselection:`Layers –> Add Delimited Text Layer`를 클릭하십시오.

  1. Create a Layer from a Delimited Text File 다이알로그에서 Browse 을 클릭하고 다운로드한 텍스트파일의 경로를 찾습니다. :guilabel:`File format`에서 :guilabel:`Custom delimiters`를 선택하고 :guilabel:`Tab`을 체크합니다. :guilabel:`Geometry definition`에서는 적절한 X, Y좌표가 발견되면 자동적으로 좌표를 불러옵니다. 여기서는 *LONGITUDE*와 *LATITUDE*입니다. 만약 잘못된 필드를 불러들이면 변경을 할 수 있습니다. :guilabel:’OK’를 클릭합니다.

참고

X와 Y의 좌표를 혼동하기 쉽습니다. 위도 Latitude는 점의 남-북위치를 표시하고 **Y**좌표가 됩니다. 마찬가지로 경도 Longitude는 점의 동-서위치를 표시하고 **X**좌표가 됩니다.

  1. 다음 다이알로그에서 에러메세지를 접하게 됩니다. 이 에러는 주로 X, Y필드가 누락된 것에 기인합니다. 원 데이터에서 에러를 살펴보고 수정할 수 있습니다. 여기서는 에러를 무시하도록 하겠습니다.

  1. 다음으로 :guilabel:`Coordinate Reference System Selector`에서 좌표계 CRS를 선택하게 됩니다. 지진 좌표는 위도와 경도이므로 `WGS 84’를 선택하고 :guilabel:`OK’를 클릭합니다.

  1. 그러면 데이터가 불러들여져서 QGIS캔버스에 표시되는 것을 볼 수 있을 것입니다.
